# 🩺 AI Medical Report Explainer

**"Understand Your Medical Reports in Simple Language"**

An AI-powered web application that lets users upload blood test reports, pathology reports, CBC, lipid profiles, liver/kidney function tests, thyroid panels, vitamin panels, and diabetes reports — and get a clear, simple-language explanation of what everything means.

> ⚠️ **Disclaimer:** This explanation is generated by AI and is for educational purposes only. It is **NOT medical advice**. Please consult a qualified doctor.

## ✅ Requirements

- Python 3.10+
- pip
- Tesseract OCR installed on the system (required by `pytesseract`)
  - **Windows:** Download installer from https://github.com/UB-Mannheim/tesseract/wiki
  - **macOS:** `brew install tesseract`
  - **Linux (Debian/Ubuntu):** `sudo apt install tesseract-ocr`
- A free Google Gemini API Key (see below)
- A modern web browser (Chrome/Edge recommended for the Speech Synthesis voice feature)

---

## 🔑 How to Get a Gemini API Key

1. Go to [Google AI Studio](https://aistudio.google.com/app/apikey).
2. Sign in with your Google account.
3. Click **"Create API Key"**.
4. Select or create a Google Cloud project when prompted.
5. Copy the generated key.
6. Paste it into your `.env` file as shown below.

## ⚙️ Installation

```bash
# 1. Clone / unzip the project
cd medical-report-explainer

# 2. Create a virtual environment
python -m venv venv

# 3. Activate it
# Windows:
venv\Scripts\activate
# macOS/Linux:
source venv/bin/activate

# 4. Install backend dependencies
pip install -r requirements.txt

# 5. Configure environment variables
# Create a .env file inside backend/ (see template below)
```

### `.env` template

```env
GEMINI_API_KEY=your_gemini_api_key_here
JWT_SECRET_KEY=replace_with_a_long_random_secret
JWT_ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=60
DATABASE_URL=sqlite:///./medical_reports.db
MAX_UPLOAD_MB=20
```

## ▶️ How to Run

### Start the backend (FastAPI)

```bash
cd backend
uvicorn main:app --reload --port 8000
```

The API will be available at: `http://127.0.0.1:8000`
Interactive API docs: `http://127.0.0.1:8000/docs`

### Open the frontend

The frontend is static HTML/CSS/JS — no build step required.

```bash
cd frontend
# Simplest option: use Python's built-in server
python -m http.server 5500
```

Then open `http://127.0.0.1:5500/index.html` in your browser.

> Make sure the `API_BASE_URL` in `frontend/js/main.js` points to your backend (default: `http://127.0.0.1:8000`).

## 🧩 Core Features

- 🔐 JWT-based Register / Login / Logout
- 📤 Drag & drop PDF upload with progress bar (max 20MB)
- 🔍 Smart OCR — automatically detects digital vs. scanned PDFs
- 🤖 Gemini-powered structured medical analysis (JSON output)
- 🎨 Color-coded results (Green = Normal, Yellow = Borderline, Red = Abnormal)
- 📊 Health Score, Risk Level, Charts (Chart.js)
- 🌐 One-click English ⇄ Marathi translation
- 🔊 "Read Aloud" explanations via Web Speech Synthesis API
- 📥 Export summary as PDF or JSON
- 🕒 Report history with search & delete
- 🌙 Dark mode toggle
- 🧠 Extra AI tools: medical term dictionary, BMI calculator, diabetes/heart risk estimators, personalized diet tips, weekly health tips

## 🔒 Security Notes

- Passwords hashed with `passlib` (bcrypt)
- JWT-protected routes for all user-specific data
- Upload validation: PDF-only, 20MB size limit
- Basic input sanitization on all endpoints
- CORS configured for local frontend origin
- Rate limiting on auth & upload routes
